Forward, backward, and stepwise regression

Too many choices!

So far, we’ve been building models to understand whether the is a relationship between some predictor variables (\(x_1, x_2, ... x_n\)) and a response variable, \(y\).

In experiments, where we only manipulate a couple of variables at a time, the structure of our models (i.e., which variables we include) is obvious: we typically include them all - that’s the point of the experiment!

However, in observational studies, we often have many variables that we could use to predict our response variable of interest. This creates a challenge!

Ideally, we would build our models using some a priori knowledge and structured hypotheses of how we think the system works, but sometimes we don’t have this knowledge (e.g., in new systems) or we’re interested in maximizing the predictive capacity of our model (i.e., how much variance, \(R^2\) we can explain)

So, how do we decide on a good fitting model?

Option 1: gotta fit ’em all!

If the number of candidate variables is small (e.g., \(k ≤ 4\)), fit all of your possible models and compare them using some pre-specified criterion (e.g., Adj-\(R^2\)): Total of \(2^k\) candidate models.

# Want to know whether abundance of bumble bees is best predicted by:
# temperature, precipitation, or whether or not a prairie was burned

m1 <- lm(abundance ~ 1) # intercept only model (null model) adj-R2 = 0.04
m2 <- lm(abundance ~ temp) # adj-R2 = 0.31
m3 <- lm(abundance ~ precip) # adj-R2 = 0.08
m4 <- lm(abundance ~ burn_status) # adj-R2 = 0.13
m5 <- lm(abundance ~ temp + precip) # adj-R2 = 0.35
m6 <- lm(abundance ~ temp + burn_status) # adj-R2 = 0.48
m7 <- lm(abundance ~ precip + burn_status) # adj-R2 = 0.19
m8 <- lm(abundance ~ temp + precip + burn_status) # adj-R2 = 0.56

Or, if we have more than 4 variables, we can conduct a more structured model selection

Option 2: backward elimination

The name sort of says it all here! In backward elimination, we start with a full model containing all of the predictors of interest (a global model), and then sequentially work to eliminate (prune) ‘unhelpful’ predictors from our global model.

  1. Fit the full model with all of our \(x_i\) of interest
  2. Find the variable with the smallest t-statistic (i.e., highest p-value) using the model summary output
  3. Fit the a new subset model with the variable omitted
  4. Check the t-stats/p-values as in Step 2 and continue until no variables have a t-stat above your threshold/p-values above your alpha (typically \(\alpha=0.05\))

Tip

We typically do backward elimination when: (1) you believe most predictors are biologically relevant and (2) you have a TON of data to support fitting a big, global model

Option 2: forward selection

The name sort of says it all here, too! In forward selection, we start with a model containing only the intercept (a null model), and then sequentially add predictors to build up to a more complex model

  1. Find the predictor \(x_i\) with the highest correlation with \(y\) (i.e., highest \(R^2\) from simple linear regressions)
  2. Fit a model with this predictor, \(x_1\), and check the T-statistic
  • if the largest t-statistic is small, stop and the best model has only an intercept, \(\beta_0\)
  • if the largest t-statistic is large, go to step 3.
  1. Find the next \(x_i\) that causes the largest increase in \(R^2\) while keeping the first \(x_1\) in the model. That is, look at ALL of the models that have both \(x_1\) plus one more and select the new variable that has the largest t-statistic.
  2. Check the largest t-statistic. Stop if it’s small, keep the variable and continue if it’s big (≥ 2).

Tip

We typically do forward selection when: (1) you suspect only a few variables matter; (2) you have a TON of candidate variables; and (3) you can’t possibly fit a global model given your sample size relative to number of candidate variables

Option 2: stepwise selection

In stepwise selection, we hybridize both backward elimination and forward selection, starting with forward selection:

  1. Add variables in the same manner you would forward selection (start with intercept only model, and add 1 variable that is best predictive of your response)
  2. After each addition, conduct backward elimination to see if any variables should be removed
  3. Continue until no additional variables can either be added or subtracted.

Caveats and warnings

There are many risks when performing model selection using these techniques, and neither approach can fully avoid them.

Backward elimination risks eliminating biologically meaningful variables

Forward selection has the potential of stopping short and not including biologically meaningful variables

Forward and backward often settle on different models

Stepwise selection only considers best immediate choice, so can converge on a local optima rather than a global optima.

Stepwise selection can bias standard error down and therefore inflate p-values, leading to more type-1 errors (false positive)

What I recommend:

In general, I counsel avoiding stepwise selection, especially when you have biologically meaningful predictors and you care about statistical inference. There are a few recommendations that have become commonplace in the biological/ecological literature that are worth heeding if you are considering doing model selection in this manner:

  1. Start with a pre-specificed set of predictor variables or models that have biological relevance, that is you have specific hypotheses as to why these variables matter.
  2. Use AIC, AICc (information criteria) rather than p-values to prune or add to models
  3. Consider model averaging (rather than relying on a single “optimal” model)
  4. Use selection approaches mostly for prediction or data exploration rather than inference

General principles and suggestions

  1. Parsimony! The best model is the simplest possible model that best explains variation in your data.
  2. Let science (experimental design, bias accounting) and common sense guide your approach
  3. Sometimes there isn’t a single best model; all models are wrong, some are useful.
  4. We may not have enough information in the data to tell what the truth is exactly!
